# 舔舐二階段處理腳本分析

## 1. 腳本定位
腳本：cat_monitoring_system/measure_ear_distance_single_video.py

本腳本屬於「第二階段分析」，前提是第一階段已判定該時段為 lick。
它的任務是把 lick 從單一類別，細分為具體舔舐部位，並輸出可量化指標。

---

## 2. 核心目標

1. 判斷舔舐命中區域（BODY_CENTER / FL / FR / HL / HR）
2. 輸出逐幀結果與事件級統計，支援後續研究分析
3. 提供可視化疊圖，便於人工驗證與除錯

---

## 3. 方法流程

### 3.1 姿態輸入與前處理
- 由 KeypointDetector 取得 17 關鍵點與信心值
- 可選 EMA 平滑座標（EMA_ALPHA < 1.0），降低單幀抖動
- 依 chest（KP 3）與 hip（KP 5）建立 body 軸與 body 長度

### 3.2 幾何區域建立
- 身體中心區：**橢圓（oriented ellipse）**，寬為 body_normal 方向、高為 body_axis 方向，半徑依 body_len 等比縮放
- 四肢區（FL/FR/HL/HR）：
  - 腳尖圓區（paw circle）：以 PAW 關節為圓心
  - 膝-腳尖長條區（knee-paw strip）：從膝關節到腳尖圓邊界前的矩形帶
- 鼻子接觸區：以鼻尖為頂部建立的**梯形（trapezoid polygon）**，而非單點；梯形寬度與厚度依 body_len 等比縮放，body_len 過短時套用短尺度補強倍率

### 3.3 命中判定策略
- 使用「**鼻子梯形與各區域交集（polygon intersection）**」決定命中，不使用橢圓
- 若同時命中多區域，採最近區域（最短 nose-to-region 距離）作主標籤（nearest_target_label）
- 四肢子區（PAW / STRIP）命中一律歸一化為肢別標籤（FL/FR/HL/HR），避免子區切換重複計數
- FRONT_VIEW_GUARD 啟用時（貓面向鏡頭導致 body_len 退化），該幀略過命中判定

### 3.4 方向向量的角色
- 由雙耳中點到鼻子的向量建立 head_dir 與延伸射線（ray_end）
- 用於可視化和診斷（判讀頭朝向、繪製箭頭）
- **不直接參與最終命中決策**，命中僅以鼻子梯形接觸幾何為準

### 3.5 事件級計數（entry count）
- target_entry_count：鼻子由 NO_TARGET 進入 BODY_CENTER 的次數
- limb_entry_count_fl/fr/hl/hr：鼻子進入對應肢區的次數
- 目的：避免只看逐幀，改看可解釋事件量

### 3.6 連續舔舐段（bout）追蹤
- 每段連續命中同一區域視為一個 bout
- 統計各區域的 bout_count 與 mean_bout_sec
- 影片結束時自動收斂尚未關閉的 bout

## 4. 輸出欄位與可用指標

### 4.1 逐幀 CSV（left_right_ear_distance.csv）
| 欄位 | 說明 |
|------|------|
| distance_px, distance_norm | 雙耳像素距離 / 正規化耳距（除以 body_len） |
| lick_zone | 命中區域：NO_TARGET / BODY_CENTER / FL / FR / HL / HR |
| lick_axis_score, lick_lateral_score | BODY_CENTER 命中時的縱向/橫向正規化座標 |
| nearest_target_label, nearest_target_t | 最近命中標籤 / 鼻子到該區距離（px） |
| limb_hit_any, limb_hit_labels | 是否命中四肢 / 命中標籤清單 |
| limb_hit_fl/fr/hl/hr_frame | 當幀四肢命中旗標（0/1） |
| limb_entry_count_fl/fr/hl/hr | 累積進入次數（截至當幀） |
| target_entry_count | 累積進入 BODY_CENTER 次數 |
| lick_time_body/fl/fr/hl/hr_sec | 累積舔舐時長（秒） |
| lick_sec_per_hit_* | 每次進入平均舔舐秒數 |
| lick_pref_pct_* | 舔舐時間偏好占比（%） |
| gaze_forward_norm, gaze_lateral_norm, gaze_angle_deg | 本體座標下的頭部方向量 |
| face_state_cat, face_state_raw, face_state | 貓咪朝向（本體計算 / 使用者規則 / 平滑後） |
| state_stability | 狀態多數決穩定度（0–1） |
| nose_detected | 當幀鼻子是否可用（1/0） |
| valid | 當幀雙耳距離是否有效（1/0） |

### 4.2 摘要 CSV（left_right_ear_distance_summary.csv）
每支影片輸出一列，欄位包含：
- processed_frames, video_elapsed_sec
- total_lick_time_sec, dominant_pref_zone, dominant_pref_pct
- 各區域：hits, lick_time_sec, pref_pct, bout_count, mean_bout_sec

## 6. 指標解讀

1. 幾何可用性高：valid 與 nose_detected 比率高，表示二階段輸入品質穩定
2. 部位判讀已可運作：可穩定區分 BODY_CENTER 與 FL 相關區域
3. 樣本分布不均：FR/HL/HR 幾乎未命中，暫不適合做泛化結論
4. 事件級計數可用：可從「逐幀命中」提升到「行為事件」分析
5. bout 統計已可用：mean_bout_sec 可作為舔舐持續性指標

## 7. 其他功能

### 7.1 多影片與 IP 串流
- VIDEO_LIST 支援本地影片路徑
- VIDEO_PATH 可遞迴掃描整個資料夾
- 支援 RTSP/HTTP IP 攝影機 URL

### 7.2 顯示模式（鍵盤控制）
| 鍵 | 功能 |
|----|------|
| q | 離開 |
| 空白 | 暫停/繼續 |
| i | 切換資訊面板顯示 |
| m | 切換模式（完整功能 / 僅 YOLO 骨架） |
| +/- | 視窗放大/縮小 |
| 1/2 | 上一支/下一支影片 |
| r | 重置當前影片 |

### 7.3 front_view_guard 機制
當貓正面對鏡頭時，body_len（胸-臀距離）因透視壓縮而失真，此時啟用守門機制（FRONT_VIEW_GUARD_ENABLED），自動跳過該幀的命中判定以避免誤判。

---

## 8. 限制與改進方向

1. 命中幀比例偏低，需擴充 lick 片段與多場景資料
2. 建議增加人工標註對照，回報部位級 precision / recall / F1
3. 可把 head_dir 衍生為方向一致性分數，作為次要輔助特徵
4. 補充時序穩定指標：區域切換率、最短事件長度、抖動率

---

## 9. 匯報用結論
本腳本已完成二階段核心任務：在已確定 lick 的前提下，將行為細分到具體身體部位並輸出可量化證據（per-frame CSV + summary CSV + bout 追蹤）。現階段重點在擴充資料與建立標註驗證，將可用原型推進到可發表的部位級評估結果。
