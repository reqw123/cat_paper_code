# 貓咪行為分析系統：架構、研究理念與目前進度

## 1. 系統定位
本系統目標是建立一個「可解釋、可量化、可擴充」的貓咪行為分析流程，從單純姿態辨識進一步提升到：
- 行為層級判斷（walk / lying / lick / shake）
- 部位層級判讀（舔舐目標區域）
- 可供論文使用的結構化指標輸出（CSV + 圖表 + 可視化）

系統設計採「兩階段思路」：
1. 第一階段：辨識是否為 lick（或其他行為）
2. 第二階段：若確定為 lick，再判斷舔舐目標部位

---

## 2. 研究理念

### 2.1 以幾何可解釋性補強純分類模型
僅靠分類結果，無法回答「為什麼」與「舔到哪裡」。
本系統加入幾何規則，讓輸出能對應到具體身體區域與軌跡。

### 2.2 以身體座標系提高泛化
透過 chest-hip 定義 body axis，將區域尺寸與座標轉換綁定到 body length，而非固定像素。
好處是不同距離、視角、體型下較穩定。

### 2.3 以時間穩定邏輯降低抖動
單幀結果容易跳動，因此加入 EMA 與多幀狀態平滑策略，降低瞬間誤判。

---

## 3. 系統架構（概念分層）

1. 感知層：YOLO pose 關鍵點抽取（17 點骨架）
2. 幾何層：身體座標系、區域定義、方向向量、接觸幾何
3. 行為層：ST-GCN 與幾何規則並行輸出
4. 事件層：連續幀整合成事件級統計（entry count、持續度）
5. 匯出層：CSV / 報告圖表 / overlay 影片

## 5. 主要衡量指標（KPI）

### A. 幾何可用性
- valid rate
- nose_detected rate

### B. 部位判讀能力
- target hit rate（lick_zone 非 NO_TARGET 比例）
- 各部位命中分布（BODY_CENTER / FL / FR / HL / HR）

### C. 事件級穩定度
- target_entry_count
- limb_entry_count_fl / fr / hl / hr
- 狀態平滑後一致性（state_stability）

### D. 泛化與可靠度（下一階段重點）
- 跨影片、跨場景、跨貓個體一致性
- 部位標註對照下的 precision / recall / F1

---

## 6. 目前風險與限制

1. 命中樣本偏少，且目前以 BODY_CENTER 與 FL 為主
2. 多視角與遮擋條件下仍需更多驗證
3. 指標以內部資料為主，尚需外部測試集與人工標註對照

---

1. 擴充 lick 片段資料量，補足 FR / HL / HR 樣本

3. 補齊穩定度指標：切換頻率、每事件持續幀數、抖動率
4. 整理論文結果章節所需表格與可視化圖
