# 渐进式姿势识别开发指南

从单一姿势到多姿势分类的完整路线图

## 📌 现阶段：二分类（推荐起点）

### 为什么从二分类开始？
- ✅ 数据需求少（每类100个序列即可）
- ✅ 训练快速（10-20分钟）
- ✅ 效果明显（准确率容易达到90%+）
- ✅ 快速验证技术可行性
- ✅ 专注于最重要的姿势

### 使用场景示例
1. **异常行为检测**：正常 vs 异常姿势
2. **特定动作识别**：正常 vs 跳跃
3. **健康监控**：正常 vs 疼痛姿势
4. **训练辅助**：正常 vs 目标动作

---

## 🚀 二分类快速上手

### Step 1: 数据收集 (30-60分钟)
```bash
python collect_binary_pose.py
```

**收集策略：**
- 每个类别收集 **100-200个序列**
- 保持两类数量**大致平衡** (比例不超过 2:1)
- 包含不同角度、光照、背景
- 目标姿势要清晰、有代表性

**操作：**
1. 播放包含目标姿势的视频
2. 看到目标姿势时：按 `1` → 按 `空格` 开始录制 → 按 `S` 保存
3. 看到正常姿势时：按 `0` → 按 `空格` 开始录制 → 按 `S` 保存
4. 重复直到各收集100+个
5. 按 `W` 写入文件

### Step 2: 训练模型 (10-20分钟)
```bash
python train_binary_pose.py
```

**输出：**
- `checkpoints_binary/best_binary_model.pth` - 最佳模型
- `checkpoints_binary/training_results.png` - 完整结果图表
  - Loss曲线
  - Accuracy曲线
  - 混淆矩阵
  - ROC曲线
  - 概率分布

### Step 3: 实时推理
```bash
python inference_binary_pose.py
```

**效果：**
- 实时显示：正常 / 目标姿势
- 置信度百分比
- 概率条形图

## 📈 扩展到3-4类（中期目标）

当二分类效果稳定（准确率 > 90%）后，可以扩展：

### 扩展策略 A: 增量添加
```python
# 修改 collect_binary_pose.py 中的类别定义
POSE_CLASSES = {
    '0': 'normal',
    '1': 'jumping',      # 原目标姿势
    '2': 'grooming',     # 新增
    '3': 'stretching',   # 新增
}
```

**优点：** 保留已有数据，只需补充新类别  
**数据需求：** 每个新类别 100+ 序列

### 扩展策略 B: 重新定义类别
根据实际需求，重新设计类别体系：

**示例1 - 活动强度分类：**
- 静止 (sitting, lying)
- 缓慢移动 (walking, grooming)
- 快速移动 (running, jumping)

**示例2 - 行为意图分类：**
- 休息
- 玩耍
- 警戒
- 捕猎

### 训练3-4类模型
```python
# 修改 train_binary_pose.py
config.num_classes = 4  # 改为实际类别数

# 使用稍大的模型
model = CatPoseLSTM(  # 改用标准LSTM
    hidden_size=128,
    num_layers=2,
    num_classes=4
)
```

**数据需求：** 每类 150-200个序列  
**训练时间：** 20-40分钟

## 💡 实用建议

### 数据收集技巧

#### ✅ 好的做法
- 每个姿势持续 1-2秒（30-60帧）
- 包含进入和退出姿势的过渡帧
- 多角度采集（左侧、右侧、正面）
- 不同猫咪个体（如果可能）
- 清晰的关键点检测

#### ❌ 避免的问题
- 姿势切换过快（序列混淆）
- 关键点遮挡严重
- 过度相似的序列（过拟合）
- 类别不平衡（比例 > 3:1）

### 模型选择建议

| 数据量 | 类别数 | 推荐模型 | 参数 |
|--------|--------|----------|------|
| < 500 | 2 | SimpleLSTM | hidden=64, layers=1 |
| 500-1000 | 2-4 | SimpleLSTM | hidden=128, layers=2 |
| 1000-2000 | 4-6 | CatPoseLSTM | hidden=128, layers=2 |
| > 2000 | 6-8 | BiLSTM | hidden=128, layers=2 |

### 性能优化

#### 提升准确率
1. **数据质量 > 数量**：宁少勿滥
2. **数据增强**：噪声、遮挡、时间扭曲
3. **类别平衡**：使用类别权重
4. **超参数调优**：learning rate, hidden size
5. **集成学习**：训练多个模型投票

#### 提升速度
1. 减小 `hidden_size` (128 → 64)
2. 减少 `num_layers` (2 → 1)
3. 使用 SimpleLSTM
4. FP16推理（已启用）

## 🛠️ 常见问题

### Q1: 二分类准确率只有70%？
**可能原因：**
- 数据量不足 (< 100/类)
- 类别定义不清晰
- 数据不平衡严重
- 序列长度不合适

**解决方案：**
- 补充数据至 150+/类
- 重新审视类别定义
- 使用类别权重
- 调整 SEQUENCE_LENGTH (试试 20 或 40)

### Q2: 如何判断可以扩展到更多类别？
**检查清单：**
- ✅ 二分类准确率 > 90%
- ✅ ROC AUC > 0.95
- ✅ 实时推理稳定无抖动
- ✅ 有足够时间收集更多数据

### Q3: 目标姿势出现频率低怎么办？
**策略：**
1. 专门录制包含目标姿势的视频
2. 使用类别权重平衡
3. 考虑异常检测而非分类
4. 数据增强增加样本
